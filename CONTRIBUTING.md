# Contributing Guide

Thank you for your interest in contributing to this repository! 🎉

Contributions are welcome and appreciated. Whether you're fixing a bug, improving documentation, optimizing an algorithm, or adding alternative solutions, your help is valuable.

---

## How to Contribute

### 1. Fork the Repository

Click the **Fork** button at the top-right corner of this repository to create your own copy.

---

### 2. Clone Your Fork

```bash
git clone https://github.com/your-username/1979-Find-Greatest-Common-Divisor-of-Array.git
```

Navigate to the project directory:

```bash
cd 1979-Find-Greatest-Common-Divisor-of-Array
```

---

### 3. Create a New Branch

Create a branch for your changes.

```bash
git checkout -b feature/your-feature-name
```

Example:

```bash
git checkout -b improve-readme
```

---

### 4. Make Your Changes

You can contribute by:

- Improving code readability
- Optimizing the algorithm
- Adding alternative solutions
- Enhancing documentation
- Fixing typos
- Improving formatting
- Updating examples
- Correcting complexity analysis

---

### 5. Test Your Changes

Before submitting, ensure that:

- The solution compiles successfully.
- The code passes all LeetCode test cases.
- No unnecessary files are added.
- Documentation is accurate.

---

### 6. Commit Your Changes

Write a clear commit message.

```bash
git add .
git commit -m "Improve README documentation"
```

---

### 7. Push Your Branch

```bash
git push origin feature/your-feature-name
```

---

### 8. Open a Pull Request

Go to your fork on GitHub and click **New Pull Request**.

Please include:

- A clear description of your changes
- Why the changes are useful
- Screenshots (if documentation/UI is affected)

---

# Contribution Guidelines

Please follow these standards:

- Use meaningful variable names.
- Keep code clean and readable.
- Follow Java coding conventions.
- Avoid unnecessary complexity.
- Include comments only when they improve understanding.
- Keep documentation professional and up to date.

---

# Code Style

- Language: Java
- Indentation: 4 spaces
- Follow standard Java naming conventions.
- Keep methods focused on a single responsibility.

---

# Reporting Issues

If you find a bug or have a suggestion:

1. Check existing issues first.
2. Open a new issue if needed.
3. Provide a clear description.
4. Include steps to reproduce the issue.
5. Share expected and actual behavior.

---
