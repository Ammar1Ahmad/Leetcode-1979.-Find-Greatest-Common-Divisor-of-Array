# Security Policy

## Supported Versions

Security updates are provided for the latest version of this repository.

| Version | Supported |
| -------- | :-------: |
| Latest   | ✅ |
| Older Versions | ❌ |

---

## Reporting a Vulnerability

The security of this project is important. If you discover a security vulnerability, please report it responsibly.

### How to Report

Please include the following information in your report:

- A clear description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact of the vulnerability.
- Suggested mitigation (if available).
- Any relevant screenshots or logs.

---

## Response Process

Once a vulnerability report is received, the following process will be followed:

1. Acknowledge receipt of the report.
2. Investigate and validate the issue.
3. Assess the severity and impact.
4. Develop and test a fix.
5. Release the fix if necessary.
6. Credit the reporter (if they wish to be acknowledged).

---

## Responsible Disclosure

Please:

- **Do not** publicly disclose the vulnerability before it has been reviewed and addressed.
- Give sufficient time for investigation and remediation.
- Report issues in good faith.

---

## Scope

This repository is intended for educational purposes and contains Java solutions for LeetCode problems.

The following are considered in scope:

- Vulnerabilities affecting repository integrity.
- Malicious code submissions.
- Dependency-related security concerns (if dependencies are introduced in the future).
- Security issues in GitHub Actions or CI/CD workflows.

The following are generally **out of scope**:

- LeetCode platform vulnerabilities.
- Compiler or JDK vulnerabilities.
- Issues caused by third-party tools or IDEs.
- General coding style or algorithmic improvements.

---

## Best Practices

Contributors are encouraged to:

- Keep dependencies up to date.
- Avoid committing sensitive information such as API keys or credentials.
- Review code before submitting pull requests.
- Follow secure coding practices.
- Use trusted development tools.

---

## Contact

If you believe you have identified a security issue, please open a **private security advisory** on GitHub (if available) or contact the repository maintainer directly through GitHub.

---

## Acknowledgments

We appreciate the efforts of security researchers and contributors who help improve the safety and quality of this repository through responsible disclosure.

---

Thank you for helping keep this project secure and reliable.
